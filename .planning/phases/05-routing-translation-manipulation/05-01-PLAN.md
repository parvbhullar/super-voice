  **Step 1: Expand types in src/redis_state/types.rs**

  Replace the existing simple RoutingRule/RoutingTableConfig with richer types. Keep the old struct names but expand fields using `#[serde(default)]` for backward compatibility:

  ```rust
  /// Match type for routing rules.
  #[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
  #[serde(rename_all = "snake_case")]
  pub enum MatchType {
      Lpm,           // Longest prefix match
      ExactMatch,    // Exact string match
      Regex,         // PCRE-style regex
      Compare,       // Comparison operators (eq, ne, gt, lt, gte, lte)
      HttpQuery,     // External HTTP API lookup
  }

  /// A target trunk with optional load distribution percentage.
  #[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
  pub struct RoutingTarget {
      pub trunk: String,
      pub load_percent: Option<u32>,  // 0-100, for weighted distribution
  }

  /// A single routing record within a routing table.
  #[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
  pub struct RoutingRecord {
      /// Match type determining how `value` is compared.
      pub match_type: MatchType,
      /// The value/pattern to match against (prefix for LPM, regex for Regex, URL for HttpQuery, etc.).
      pub value: String,
      /// Optional comparison operator for Compare match type: "eq", "ne", "gt", "lt", "gte", "lte".
      #[serde(default)]
      pub compare_op: Option<String>,
      /// Which SIP field to match against. Default is "destination_number".
      #[serde(default = "default_match_field")]
      pub match_field: String,
      /// Primary target(s) with optional load distribution.
      pub targets: Vec<RoutingTarget>,
      /// If set, jump to another routing table instead of using targets.
      #[serde(default)]
      pub jump_to: Option<String>,
      /// Priority for ordering (lower = higher priority). Default 100.
      #[serde(default = "default_priority")]
      pub priority: u32,
      /// Whether this is the default/fallback record.
      #[serde(default)]
      pub is_default: bool,
  }

  fn default_match_field() -> String { "destination_number".to_string() }
  fn default_priority() -> u32 { 100 }
  ```

  Update RoutingTableConfig to use RoutingRecord instead of RoutingRule. Keep RoutingRule as a type alias or deprecated wrapper for backward compat if needed, but the new API should use RoutingRecord. The RoutingTableConfig becomes:

  ```rust
  pub struct RoutingTableConfig {
  ```

  **Step 5: Create src/routing/engine.rs**

  Implement `RoutingEngine`:

  ```rust
  pub struct RoutingEngine {
      config_store: Arc<ConfigStore>,
      http_client: reqwest::Client,
  }

  /// Result of a routing resolution.
  pub struct RouteResult {
      pub trunk: String,
      pub matched_record: Option<RoutingRecord>,
      pub table_name: String,
  }

  impl RoutingEngine {
      pub fn new(config_store: Arc<ConfigStore>) -> Self;

      /// Resolve a routing decision for the given call context.
      pub async fn resolve(
          &self,
          table_name: &str,
          context: &RouteContext,
      ) -> Result<Option<RouteResult>>;
  }

  pub struct RouteContext {
      pub destination_number: String,
      pub caller_number: String,
      pub caller_name: Option<String>,
      // Extensible: SIP headers can be added later
  }
  ```

  The resolve() method:
  1. Load the routing table from ConfigStore
  2. Sort records by priority (lower first), with is_default records last
  3. For each record, evaluate the match based on match_type:
     - **Lpm**: Check if destination starts with record.value; track longest match across all LPM records
     - **ExactMatch**: destination == record.value
     - **Regex**: regex::Regex::new(record.value).is_match(field_value)
     - **Compare**: Parse both sides as numbers or strings, apply compare_op (eq/ne/gt/lt/gte/lte)
     - **HttpQuery**: Call http_query_lookup with record.value as URL
  4. If matched record has `jump_to`, recursively resolve on the target table with depth tracking
  5. Max recursion depth = 10. At depth 11, return `Err("routing loop: max depth 10 exceeded")`
  6. If matched record has targets, select using weighted distribution (load_percent)
  7. If no record matches, check for is_default record
  8. Return RouteResult or None

  For weighted target selection: if a record has multiple targets with load_percent, pick one proportionally (similar to select_gateway in distribution.rs but for RoutingTarget).
