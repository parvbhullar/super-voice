Create PjFailoverLoop as specified in design doc Phase 3, Task 3.2.

1. Create `src/proxy/pj_failover.rs`:
   - PjFailoverResult enum: Connected{gateway_addr, call_event_rx, sdp, call_id}, NoFailover{code, reason}, Exhausted{last_code, last_reason}, NoRoutes
   - PjFailoverLoop { dialog_layer: PjDialogLayer, cancel_token: CancellationToken }
   - PjFailoverLoop::new(dialog_layer, cancel_token) -> Self
   - try_routes(&self, trunk: &TrunkConfig, caller_sdp, caller_uri, callee_uri) -> Result<PjFailoverResult>:
     - Return NoRoutes if gateways empty
     - Build PjCredential from trunk.credentials if present
     - For each gateway: construct target URI as sip:{user}@{gateway.name}, call dialog_layer.create_invite(), wait_for_outcome()
     - On Connected: return immediately
     - On NoFailover: return immediately (do not try next)
     - On Failed: record last code/reason, continue to next gateway
     - After all exhausted: return Exhausted
   - wait_for_outcome(event_rx, trunk, gateway_name) -> WaitOutcome:
     - Loop receiving PjCallEvent from event_rx with 30s timeout
     - tokio::select! with cancel_token.cancelled() -> Failed{487, "Cancelled"}
     - Trying: continue
     - Ringing: continue
     - EarlyMedia: store sdp as early_media_sdp, continue
     - Confirmed{sdp}: return Connected, use early_media_sdp as fallback if sdp is None
     - Terminated{code, reason}: check is_nofailover(code, trunk), return NoFailover or Failed
     - Timeout: return Failed{408, "Request Timeout"}
   - WaitOutcome enum (private): Connected{sdp, call_id}, NoFailover{code, reason}, Failed{code, reason}
   - extract_user(uri) helper: strip sip:/sips: prefix, take part before @

2. Update `src/proxy/mod.rs`:
   Add:
