Create `src/proxy/pj_failover.rs` (gated by carrier feature):

PjFailoverResult enum:
- Connected { gateway_addr: String, call_event_rx: PjCallEventReceiver, sdp: Option<String>, call_id: String }  // call_id from PjCallEvent::Confirmed
- NoFailover { code: u16, reason: String }
- Exhausted { last_code: u16, last_reason: String }
- NoRoutes

PjFailoverLoop struct { dialog_layer: PjDialogLayer, cancel_token: CancellationToken }.

PjFailoverLoop::new(dialog_layer, cancel_token) -> Self.

PjFailoverLoop::try_routes(&self, trunk: &TrunkConfig, caller_sdp: &str, caller_uri: &str, callee_uri: &str) -> Result<PjFailoverResult>:
- Return NoRoutes if gateways empty
- Build PjCredential from trunk.credentials if present
- For each gateway: build target_uri as `sip:{user}@{gateway_ref.name}`, call dialog_layer.create_invite(), wait_for_outcome()
- On Connected: return with call_id from Confirmed event
- On NoFailover: return immediately
- On Failed: continue to next gateway
- After all fail: return Exhausted

wait_for_outcome(&self, event_rx, trunk, gateway_name) -> WaitOutcome:
- Track early_media_sdp for fallback
- 30s timeout per gateway
- select! on cancel_token + timeout(event_rx.recv())
- Match events:
  - Trying/Ringing: continue
  - EarlyMedia: save sdp, continue
  - Confirmed { call_id, sdp }: return Connected with call_id and sdp.or(early_media_sdp)
  - Terminated { code, reason }: check is_nofailover(code, trunk) from crate::proxy::failover::is_nofailover, return NoFailover or Failed

Private WaitOutcome enum: Connected{sdp, call_id}, NoFailover{code, reason}, Failed{code, reason}.

Private extract_user(uri) helper: strip sip:/sips: prefix, take part before @.

Add to `src/proxy/mod.rs`:
```rust
#[cfg(feature = "carrier")]
pub mod pj_failover;
```

Reference `docs/plans/2026-04-01-pjsip-migration.md` Phase 3 Tasks 3.1 and 3.2 for exact implementations. Key fix vs design doc: Confirmed event now carries call_id, so wait_for_outcome extracts it directly instead of the TODO placeholder.
