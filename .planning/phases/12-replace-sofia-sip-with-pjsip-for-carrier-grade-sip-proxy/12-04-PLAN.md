2. Create `src/endpoint/pjsip_endpoint.rs`:
   - PjsipEndpoint struct { name: String, config: EndpointConfig, bridge: Option<Arc<PjBridge>>, running: Arc<AtomicBool> }
   - PjsipEndpoint::from_config(config: &EndpointConfig) -> Result<Self>: Accept stack="pjsip" or "sofia" (backward compat). Return error for other stack names.
   - PjsipEndpoint::bridge(&self) -> Option<Arc<PjBridge>>: Return clone of bridge Arc. This is the API that AppState uses to get the bridge reference (RESEARCH GAP 4).
   - impl SipEndpoint for PjsipEndpoint:
     - name() -> &self.name
     - stack() -> "pjsip"
     - listen_addr() -> format bind_addr:port
     - start(): Build PjEndpointConfig from EndpointConfig (map bind_addr, port, transport, tls fields; session_timers=true, session_expires=1800, min_se=90, enable_100rel=true). Call PjBridge::start(pj_config), store Arc in self.bridge, set running=true.
     - stop(): Send PjCommand::Shutdown to bridge, take bridge from Option, set running=false.
     - is_running(): load from AtomicBool

3. In `src/endpoint/mod.rs`:
   - Replace `pub mod sofia_endpoint;` with `pub mod pjsip_endpoint;` (both under `#[cfg(feature = "carrier")]`)
   - Replace `pub use sofia_endpoint::SofiaEndpoint;` with `pub use pjsip_endpoint::PjsipEndpoint;`

4. In `src/endpoint/manager.rs`:
   - Replace `use crate::endpoint::SofiaEndpoint;` with `use crate::endpoint::PjsipEndpoint;` (under carrier cfg)
   - In create_endpoint match: replace `"sofia" => Box::new(SofiaEndpoint::from_config(config)?)` with `"sofia" | "pjsip" => Box::new(PjsipEndpoint::from_config(config)?)` (accepting both names for backward compat with existing Redis endpoint configs)

   - Practical approach: Add a `bridge_handle(&self) -> Option<Arc<dyn std::any::Any + Send + Sync>>` method to SipEndpoint trait (default returns None). PjsipEndpoint overrides to return `Some(self.bridge.clone()? as Arc<dyn Any + Send + Sync>)`. In AppState construction: downcast to `Arc<PjBridge>`.
