Create `src/security/message_validator.rs`:
- `pub struct SipMessageInfo` with fields: `max_forwards: Option<u32>`, `content_length: Option<usize>`, `body_length: usize`, `method: String`.
- `pub enum ValidationResult`: `Valid`, `InvalidMaxForwards { value: Option<u32> }`, `ContentLengthMismatch { header: usize, actual: usize }`, `MissingMaxForwards`.
- `pub fn validate_sip_message(msg: &SipMessageInfo) -> ValidationResult`:
  1. If max_forwards is None, return MissingMaxForwards.
  2. If max_forwards is Some(0), return InvalidMaxForwards.
  3. If content_length is Some(n) and n != body_length, return ContentLengthMismatch.
  4. Return Valid.
- This is a pure function operating on a simple struct, not on raw SIP bytes. The caller (endpoint/security integration) will parse headers and construct SipMessageInfo before calling.

Create `src/security/topology.rs`:
- `pub struct SipHeaders` with `headers: Vec<(String, String)>` (name-value pairs).
- `pub fn hide_topology(headers: &mut SipHeaders, internal_domains: &[&str])`:
  1. Filter Via headers: keep only the first (outermost) Via. Remove Via entries whose host matches an internal domain pattern.
  2. Filter Record-Route headers: remove entries whose URI contains an internal domain.
  3. Leave all other headers unchanged.
- `internal_domains` is a list of domain/IP patterns considered internal (e.g. "10.0.0.", "192.168.", "internal.example.com").
- Simple substring matching for internal domain detection (not regex — performance-sensitive path).
