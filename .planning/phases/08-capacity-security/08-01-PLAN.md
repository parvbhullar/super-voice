Create `src/capacity/fallback.rs`:
- `LocalCapacityFallback` struct with `DashMap<String, AtomicU64>` for CPS counters and concurrent counters per trunk.
- Methods: `increment_cps(trunk) -> u64`, `get_cps(trunk) -> u64`, `increment_concurrent(trunk) -> u64`, `decrement_concurrent(trunk)`, `get_concurrent(trunk) -> u64`.
- CPS entries expire via a background Tokio task that resets counts every 1 second (or use a simple timestamp-based check).
- Use `std::sync::atomic::AtomicU64` and `std::collections::HashMap` wrapped in `Arc<RwLock<>>` (or DashMap if available; check Cargo.toml first — if not present, use `std::sync::RwLock<HashMap<>>`).

Create `src/capacity/guard.rs`:
- `CapacityCheckResult` enum: `Allowed`, `CpsExceeded { current: u64, limit: u32 }`, `ConcurrentExceeded { current: u64, limit: u32 }`, `TrunkBlocked { until: Instant, reason: String }`.
- `CapacityGuard` struct holding `Option<Arc<RuntimeState>>`, `LocalCapacityFallback`, and a block tracker `RwLock<HashMap<String, BlockEntry>>` where `BlockEntry` has `until: Instant`, `escalation_count: u32`.
- `CapacityGuard::new(runtime_state: Option<Arc<RuntimeState>>) -> Self`.
- `async fn check_capacity(&self, trunk_name: &str, call_id: &str, config: &CapacityConfig) -> CapacityCheckResult`:
  1. Check block tracker first — if trunk is blocked and block hasn't expired, return TrunkBlocked.
  2. If runtime_state is Some, try Redis CPS check: call `record_cps_event` then `get_cps_count` with 1-second window. On Redis error, fall through to local fallback.
  3. If Redis succeeded: compare CPS count to `config.max_cps`. If exceeded, call `auto_block(trunk_name)` and return CpsExceeded.
  4. Check concurrent calls via `get_concurrent_calls`. If >= max_calls, return ConcurrentExceeded.
  5. If Redis is unavailable (None or error), use LocalCapacityFallback for both checks.
  6. Return Allowed.
- `fn auto_block(&self, trunk_name: &str)`: Insert/update block entry. Base duration 60s. If existing entry with escalation_count N, new duration = 60 * 3^N seconds, increment escalation_count. Cap at 3600s (1 hour).
- `fn is_blocked(&self, trunk_name: &str) -> Option<BlockEntry>`: Check if blocked and not expired. If expired, remove entry.
- `async fn release_call(&self, trunk_name: &str, call_id: &str)`: Decrement concurrent calls in Redis and fallback.

Add `pub mod capacity;` to `src/lib.rs`.

Important: The Lua-script atomic CPS check is NOT needed yet — the existing `record_cps_event` + `get_cps_count` on RuntimeState is sufficient since they use ZSET with millisecond timestamps. The two-step (record + count) is acceptable because slight over-admission under extreme concurrency is preferable to complexity. Document this as a TODO for Lua atomicity optimization.

In `dispatch_proxy_call`, after loading trunk config (step 2) and before applying translations (step 3), insert a capacity check:
```rust
// Step 2.5: Capacity check
if let Some(ref cap_config) = trunk.capacity {
    if let Some(ref guard) = app_state.capacity_guard {
        match guard.check_capacity(&trunk.name, &session_id, cap_config).await {
            CapacityCheckResult::Allowed => {
                info!(session_id = %session_id, "dispatch: capacity check passed");
            }
            CapacityCheckResult::CpsExceeded { current, limit } => {
                warn!(session_id = %session_id, trunk = %trunk.name,
                    current, limit, "dispatch: CPS limit exceeded — rejecting with 503");
                return Err(anyhow!("CPS limit exceeded for trunk '{}': {}/{}", trunk.name, current, limit));
            }
            CapacityCheckResult::ConcurrentExceeded { current, limit } => {
                warn!(session_id = %session_id, trunk = %trunk.name,
                    current, limit, "dispatch: concurrent call limit exceeded — rejecting with 503");
                return Err(anyhow!("concurrent limit exceeded for trunk '{}': {}/{}", trunk.name, current, limit));
            }
            CapacityCheckResult::TrunkBlocked { reason, .. } => {
                warn!(session_id = %session_id, trunk = %trunk.name,
                    reason, "dispatch: trunk auto-blocked — rejecting with 503");
                return Err(anyhow!("trunk '{}' is temporarily blocked: {}", trunk.name, reason));
            }
        }
    }
}
```

Also add a `release_call` at the end of the session (after session.run completes, whether success or error) to decrement the concurrent counter:
```rust
if let Some(ref guard) = app_state.capacity_guard {
    guard.release_call(&trunk.name, &session_id).await;
}
```
