  <action>
Add `security_module: Option<Arc<SipSecurityModule>>` to `AppStateInner` in `src/app.rs`. Initialize it in `AppStateBuilder::build()` with `SecurityConfig::default()`. Always create it (not gated on Redis — security is local-first).

Create `src/handler/security_api.rs` with 6 handlers following existing handler patterns (see trunks_api.rs for style reference):

1. `GET /api/v1/security/firewall` — `get_firewall`:
   - Extract SipSecurityModule from AppState (return 503 if None via `require_security!` macro).
   - Return JSON: `{ "whitelist": [...], "blacklist": [...], "ua_blacklist": [...] }`.

2. `PATCH /api/v1/security/firewall` — `patch_firewall`:
   - Accept JSON body with optional `whitelist`, `blacklist`, `ua_blacklist` fields.
   - Apply merge: if field present, replace; if absent, keep existing.
   - Rebuild SipSecurityModule internals (recompile regexes, reparse CIDRs).
   - Return updated config.

3. `GET /api/v1/security/blocks` — `list_blocks`:
   - Call `security_module.get_blocked_ips()`.
   - Return JSON array of `{ "ip": "...", "reason": "flood|brute_force", "blocked_until": "ISO8601", "details": {...} }`.

4. `DELETE /api/v1/security/blocks/{ip}` — `delete_block`:
   - Call `security_module.unblock_ip(ip)`.
   - Return 200 with `{ "unblocked": true }` or 404 if IP was not blocked.

5. `GET /api/v1/security/flood-tracker` — `get_flood_tracker`:
   - Return current flood tracking stats: per-IP request counts, any active blocks.
   - JSON: `{ "tracked_ips": N, "blocked_ips": N, "entries": [...] }`.

6. `GET /api/v1/security/auth-failures` — `get_auth_failures`:
   - Return brute-force tracking stats: per-IP failure counts, any active blocks.
   - JSON: `{ "tracked_ips": N, "blocked_ips": N, "entries": [...] }`.

Define a `require_security!` macro similar to `require_config_store!` used in other handler modules:
```rust
macro_rules! require_security {
    ($state:expr) => {
        match $state.security_module.as_ref() {
            Some(s) => s.clone(),
            None => return (StatusCode::SERVICE_UNAVAILABLE, Json(json!({"error": "security module not configured"}))).into_response(),
        }
    };
}
```
