  <action>
Create `src/handler/calls_api.rs` with 6 handlers:

**Response types** (Serialize):
- `CallSummary`: session_id, call_type (string), caller, callee, start_time (ISO8601), answer_time (Option), duration_secs (computed from start_time to now), status (ringing/answered/on_hold)
- `CallDetail`: all CallSummary fields + trunk_name (Option), did_number (Option), codec (Option), media_mode (relay/transcode)

**Handlers**:

1. `pub async fn list_calls(State(app_state): State<AppState>) -> impl IntoResponse`:
   - Lock active_calls, iterate entries
   - For each ActiveCallRef: read call_state, build CallSummary
   - Return Json(Vec<CallSummary>) with 200

2. `pub async fn get_call(State(app_state): State<AppState>, Path(id): Path<String>) -> impl IntoResponse`:
   - Lock active_calls, find by session_id
   - If found: read call_state, build CallDetail, return 200
   - If not found: return 404 { "error": "call not found" }

3. `pub async fn hangup_call(State(app_state): State<AppState>, Path(id): Path<String>) -> impl IntoResponse`:
   - Find call in active_calls
   - Send Command::Hangup via cmd_sender (broadcast channel)
   - Return 200 { "status": "terminating" }
   - If not found: 404

4. `pub async fn transfer_call(State(app_state): State<AppState>, Path(id): Path<String>, Json(body): Json<TransferRequest>) -> impl IntoResponse`:
   - TransferRequest struct: target: String (SIP URI to transfer to)
   - Find call, send Command::Refer { caller, callee: body.target, options: None }
   - Return 200 { "status": "transferring" }
   - If not found: 404

5. `pub async fn mute_call(State(app_state): State<AppState>, Path(id): Path<String>) -> impl IntoResponse`:
   - Find call, send Command::Mute { track_id: None }
   - Return 200 { "status": "muted" }
   - If not found: 404

6. `pub async fn unmute_call(State(app_state): State<AppState>, Path(id): Path<String>) -> impl IntoResponse`:
   - Find call, send Command::Unmute { track_id: None }
   - Return 200 { "status": "unmuted" }
   - If not found: 404

Tests: Create unit tests using the same pattern as existing handler tests (check routing_api.rs or trunks_api.rs for the TDD route test pattern). Tests should verify 404 responses for missing calls and correct JSON shape for list response.
