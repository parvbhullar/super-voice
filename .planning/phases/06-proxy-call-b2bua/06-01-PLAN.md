  <behavior>
    - Test 1: MediaBridge::new() with same codec on both legs sets needs_transcoding=false
    - Test 2: MediaBridge::new() with different codecs sets needs_transcoding=true
    - Test 3: codec_optimization selects matching codec when both peers support it
    - Test 4: RTP timestamp continuity: timestamps with >10s jump get rewritten

  </behavior>
  <action>
Implement `src/proxy/media_bridge.rs` following the pattern from `media-gateway/src/proxy/proxy_call/media_bridge.rs`:

**MediaBridge struct**:
- Fields: leg_a: Arc<dyn MediaPeer>, leg_b: Arc<dyn MediaPeer>, codec_a: CodecType, codec_b: CodecType, params_a: rustrtc::RtpCodecParameters, params_b: rustrtc::RtpCodecParameters, dtmf_pt_a: Option<u8>, dtmf_pt_b: Option<u8>, started: AtomicBool
- `pub fn new(...)` constructor
- `pub fn needs_transcoding(&self) -> bool` returns `self.codec_a != self.codec_b`
- `pub async fn start(&self) -> Result<()>`: if already started return Ok. Get PeerConnections from both peers' tracks. Spawn tokio task with `bridge_pcs` that:
  - When codecs match: forward AudioFrames directly from pc_a to pc_b and vice versa (zero-copy — move frame, don't clone payload bytes)
  - When codecs differ: decode source frame -> if sample rates differ resample (8kHz<->16kHz) -> encode to target codec -> forward
  - RTP timestamp continuity: track last_timestamp per direction; if gap > 10 seconds (80000 samples at 8kHz), rewrite timestamp to last + frame_duration
  - Duplicate detection: track last_sequence_number per direction; skip frames with same seq
- `pub fn stop(&self)`: cancel both peers

**Codec optimization** (standalone function):
- `pub fn optimize_codecs(caller_codecs: &[CodecType], callee_codecs: &[CodecType]) -> Option<CodecType>`: find first common codec preferring G.711 (Pcmu, Pcma) for zero-copy relay. Returns None if no common codec.

Adapt the `bridge_pcs` and `forward_track` methods from media-gateway but simplified: no recorder integration (that comes in Phase 9 CDR), but keep the bidirectional PeerConnectionEvent loop pattern with FuturesUnordered forwarders.

Write tests in a `#[cfg(test)] mod tests` block at the bottom of media_bridge.rs (or separate file if too large). Use mock MediaPeer implementations for unit testing codec selection and timestamp logic.
