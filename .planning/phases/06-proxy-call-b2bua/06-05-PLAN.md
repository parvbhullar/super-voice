- Use rsipstack test utilities to create a caller UAC that sends INVITE
- Verify the proxy creates outbound INVITE to the configured gateway
- Verify 200 OK returned to caller with SDP
- Verify both dialog guards are active (call is bridged)
- Verify call appears in active_calls
- Clean up: send BYE from caller side

**SC2 — Zero-copy RTP relay when codecs match** (test_proxy_call_zero_copy_relay):
- Configure both legs with G.711 (Pcmu)
- Verify MediaBridge.needs_transcoding() returns false
- Verify the bridge starts without creating transcoder instances
- This can be tested at the unit level by checking the MediaBridge state after creation

**SC3 — API visibility** (test_active_call_api_visibility):
- Start a proxy call (using the setup from SC1)
- While call is active: GET /api/v1/calls -> verify call appears with correct session_id, trunk, DID
- GET /api/v1/calls/{session_id} -> verify call detail returns 200
- POST /api/v1/calls/{session_id}/hangup -> verify call terminates
- GET /api/v1/calls -> verify empty list after hangup

Use unique trunk/DID/table names with UUID prefixes (per Phase 5 pattern) to avoid collisions with parallel tests. Each test should clean up its Redis keys.
