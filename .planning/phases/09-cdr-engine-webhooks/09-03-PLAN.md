**src/cdr/store.rs:**
- `pub struct CdrStore` with `pool: RedisPool`
- Redis key design:
  - `cdr:detail:{uuid}` — STRING, full CDR JSON (already from CdrQueue, but CdrStore manages persistence beyond the queue TTL)
  - `cdr:index:all` — SORTED SET, score=unix timestamp, member=uuid (for time-ordered listing)
  - `cdr:index:trunk:{trunk_name}` — SORTED SET, score=unix timestamp, member=uuid
  - `cdr:index:did:{did_number}` — SORTED SET, score=unix timestamp, member=uuid
  - `cdr:index:status:{status}` — SORTED SET, score=unix timestamp, member=uuid
- `pub async fn save(&self, cdr: &CarrierCdr) -> Result<()>`:
  - SET cdr:detail:{uuid} with no TTL (persistent CDR storage — or configurable TTL like 86400*30 for 30 days)
  - ZADD to all relevant index sorted sets with score = created_at.timestamp()
- `pub async fn get(&self, uuid: &str) -> Result<Option<CarrierCdr>>`:
  - GET cdr:detail:{uuid}, deserialize
- `pub async fn delete(&self, uuid: &str) -> Result<bool>`:
  - GET the CDR first to know which indexes to clean
  - DEL cdr:detail:{uuid}
  - ZREM from all relevant index sets
  - Return true if existed
- `pub async fn list(&self, filter: &CdrFilter) -> Result<CdrPage>`:
  - `CdrFilter { trunk, did, status, start_date, end_date, page, page_size }` — all optional
  - Determine which index set to scan:
    - If trunk filter: use `cdr:index:trunk:{trunk}`
    - Elif did filter: use `cdr:index:did:{did}`
    - Elif status filter: use `cdr:index:status:{status}`
    - Else: use `cdr:index:all`
  - ZREVRANGEBYSCORE with date range (or -inf/+inf), LIMIT for pagination
  - Load each CDR detail, apply any remaining filters client-side
  - Return `CdrPage { items: Vec<CarrierCdr>, total: u64, page: u32, page_size: u32 }`

- Also update `CdrProcessor` (from Plan 02) to call `cdr_store.save()` before webhook delivery — this ensures CDRs are indexed for queries. The processor should be updated to accept CdrStore in addition to CdrQueue.
