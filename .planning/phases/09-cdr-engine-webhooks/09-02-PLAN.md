**src/cdr/webhook.rs:**
- `pub async fn deliver_webhook(url: &str, secret: Option<&str>, cdr: &CarrierCdr) -> Result<()>`
  - POST CDR JSON to url with Content-Type: application/json
  - If secret is Some, add X-Webhook-Secret header
  - Add X-Webhook-Event: "cdr.new" header
  - Timeout: 5 seconds per attempt
  - On non-2xx or error: retry up to 3 times with delay of 2^attempt seconds (1s, 2s, 4s)
  - After all retries exhausted, return Err
- Use `reqwest::Client` with timeout

**src/cdr/disk_fallback.rs:**
- `pub async fn write_cdr_to_disk(fallback_dir: &str, cdr: &CarrierCdr) -> Result<String>`
  - Create directory: `{fallback_dir}/{YYYYMMDD_HH}/` (hourly rotation)
  - Write file: `{uuid}.json` containing serialized CDR
  - Return the full file path
  - Use `tokio::fs` for async I/O

**src/cdr/processor.rs:**
- `pub struct CdrProcessor` with fields: `cdr_queue: Arc<CdrQueue>`, `config_store: Arc<ConfigStore>`, `fallback_dir: String`, `cancel_token: CancellationToken`
- `pub async fn run(&self)` — loop:
  1. BRPOP-style: call `cdr_queue.dequeue()`, if None sleep 1s and retry
  2. Load all active webhooks from `config_store.list_webhooks()`
  3. For each active webhook: call `deliver_webhook(url, secret, &cdr)`
  4. If ALL webhooks fail (or no webhooks registered): call `write_cdr_to_disk(fallback_dir, &cdr)`
  5. Respect cancel_token for graceful shutdown
- `pub fn spawn(self) -> tokio::task::JoinHandle<()>` — convenience to spawn as background task
