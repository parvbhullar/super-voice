   ```rust
   pub trait SipEndpoint: Send + Sync {
       fn name(&self) -> &str;
       fn stack(&self) -> &str;  // "sofia" or "rsipstack"
       fn listen_addr(&self) -> String;  // e.g. "0.0.0.0:5060"

       fn is_running(&self) -> bool;
   }
   ```
   Export modules: manager, sofia_endpoint, rsip_endpoint.

   Also export a `validate_digest_auth` helper function (or put it in a shared `auth` submodule) that both endpoints use:
   ```rust
   pub fn validate_digest_auth(
       auth_header: &str,
       expected_username: &str,
       expected_password: &str,
       realm: &str,
       nonce: &str,
   ) -> bool
   ```
   This function parses the Digest Authorization header (extracting username, realm, nonce, uri, response fields), computes the expected digest per RFC 2617 (HA1 = MD5(username:realm:password), HA2 = MD5(method:uri), response = MD5(HA1:nonce:HA2)), and returns true only if the computed response matches the provided response. Use the `md5` crate for hashing.

2. **src/endpoint/manager.rs**: EndpointManager that owns a HashMap&lt;String, Box&lt;dyn SipEndpoint&gt;&gt; and:
   - `new()` — creates empty manager
   - `create_endpoint(config: &EndpointConfig) -> Result<()>` — dispatches to SofiaEndpoint::from_config or RsipEndpoint::from_config based on config.stack field. Stores in map. Calls start().
   - `stop_endpoint(name: &str) -> Result<()>` — calls stop() on the endpoint and removes from map
   - `stop_all() -> Result<()>` — stops all endpoints
   - `get_endpoint(name: &str) -> Option<&dyn SipEndpoint>` — lookup
   - `list_endpoints() -> Vec<&dyn SipEndpoint>` — list all active
   - `load_from_config_store(store: &ConfigStore) -> Result<()>` — loads all EndpointConfigs from Redis and creates endpoints for each

   EndpointManager should be wrapped in Arc&lt;Mutex&gt; for thread-safe access from axum handlers.

   Use `#[cfg(feature = "carrier")]` to gate Sofia-SIP endpoint creation. When carrier feature is not enabled and stack="sofia" is requested, return an error "Sofia-SIP stack requires 'carrier' feature".

1. **src/endpoint/sofia_endpoint.rs** (gated behind `#[cfg(feature = "carrier")]`):
   - `SofiaEndpoint` struct holds: name (String), config (EndpointConfig), agent (Option&lt;NuaAgent&gt;), running (AtomicBool), event_task (Option&lt;JoinHandle&gt;)
   - `from_config(config: &EndpointConfig) -> Result<Self>` — validates config.stack == "sofia", stores config
   - `SipEndpoint::start()`:
     - Build bind_url from config: `sip:{bind_addr}:{port};transport={transport}`. If transport is "tls", use `sips:` scheme and append TLS cert path parameters for Sofia-SIP.
     - Create NuaAgent::new(bind_url)
     - If config.nat is set with external_ip, configure via Sofia NUTAG parameters (document as TODO if NuaAgent API doesn't yet support NAT params — Phase 3 may need to extend NuaAgent)
     - If config.auth is set, store credentials for digest auth challenge handling
     - If config.session_timer is set, configure session timer interval
     - Spawn a tokio task that loops on agent.next_event() and handles:
       - IncomingInvite/IncomingRegister with auth configured:
         1. If NO Authorization header: respond 407 with WWW-Authenticate header containing realm (from config.auth.realm) and a generated nonce (random hex string, stored for validation)
         2. If Authorization header IS present: parse it and call validate_digest_auth() with the provided header, config.auth.username, config.auth.password, config.auth.realm, and the stored nonce. If validation PASSES: respond 200 OK. If validation FAILS: respond 403 Forbidden.
       - IncomingInvite/IncomingRegister without auth configured: respond 200
       - Other events: log at debug level
   - `SipEndpoint::stop()`: call agent.shutdown(), abort event task, set running=false

2. **src/endpoint/rsip_endpoint.rs**:
   - `RsipEndpoint` struct holds: name (String), config (EndpointConfig), running (AtomicBool), cancel_token (CancellationToken)
   - `from_config(config: &EndpointConfig) -> Result<Self>` — validates config.stack == "rsipstack"
   - `SipEndpoint::start()`:
     - Build SocketAddr from config.bind_addr and config.port
     - Create rsipstack Endpoint (follow existing pattern in src/app.rs AppStateBuilder::build)
     - If transport is "tls", configure TLS with cert/key paths
     - If config.nat.external_ip is set, use it as the Contact header address
     - If config.session_timer is set, enable session timer support
     - Store the endpoint handle for later stop
   - `SipEndpoint::stop()`: cancel the token, set running=false

   For digest auth on rsipstack: same three-step flow as Sofia:
   1. No Authorization header -> respond 407 with WWW-Authenticate (realm + nonce)
   2. Authorization header present, validate_digest_auth() passes -> accept (200)
   3. Authorization header present, validate_digest_auth() fails -> reject (403 Forbidden)

   This is the standard SIP digest auth flow per RFC 2617 / RFC 3261 Section 22.

3. **tests/endpoint_auth_test.rs**: Unit tests for the validate_digest_auth function:
   - Test with a known valid digest: compute MD5 manually for username="alice", password="secret", realm="sip.example.com", nonce="abc123", method="INVITE", uri="sip:bob@example.com". Assert returns true.
   - Test with wrong password in the computed digest: assert returns false.
   - Test with malformed header (missing fields): assert returns false.
   - Test with empty string: assert returns false.

Both implementations must implement the SipEndpoint trait.
