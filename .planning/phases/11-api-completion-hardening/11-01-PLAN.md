<task type="auto">
  <name>Task 1: Implement diagnostics and system API handlers</name>
  <files>src/handler/diagnostics_api.rs, src/handler/system_api.rs</files>
  <action>
Create `src/handler/diagnostics_api.rs` with 5 endpoint handlers:

1. **POST /api/v1/diagnostics/trunk-test** (`trunk_test`): Accept JSON `{"trunk": "name"}`. Look up TrunkConfig from config_store, iterate its gateways, for each gateway send a TCP/TLS connect probe to proxy_addr (tokio::net::TcpStream::connect with 3-second timeout). Return JSON array of `{"gateway": name, "reachable": bool, "latency_ms": u64}`. Use `require_config_store!` macro pattern. If trunk not found return 404. If gateway_manager is None return 503.

2. **POST /api/v1/diagnostics/route-evaluate** (`route_evaluate`): Accept JSON `{"source": "+1234", "destination": "+5678", "routing_table": "default"}`. Create a RouteContext from source/destination, instantiate RoutingEngine from config_store, call resolve(). If trunk has translation_classes, apply TranslationEngine::apply() to show what translations would occur. Return JSON `{"matched_route": {table, trunk, record}, "translations": [{class, before, after}], "selected_trunk": name}`. No actual call placed.

3. **GET /api/v1/diagnostics/registrations** (`list_registrations`): Read registration_handles from AppState, return JSON array of `{"user": key, "registered": true}` for each entry. Also include alive_users set.

4. **GET /api/v1/diagnostics/registrations/{user}** (`get_registration`): Look up a specific user in registration_handles. Return 404 if not found.

5. **GET /api/v1/diagnostics/summary** (`diagnostics_summary`): Return a combined summary: total gateways, healthy gateways, total registrations, total active calls.

Create `src/handler/system_api.rs` with 6 endpoint handlers:

1. **GET /api/v1/system/health** (`system_health`): Return JSON `{"status": "ok"|"degraded", "uptime_seconds": u64, "redis_connected": bool, "active_calls": u64, "total_calls": u64}`. Check Redis connectivity by attempting a PING on the pool. If Redis unreachable, status is "degraded".

2. **GET /api/v1/system/info** (`system_info`): Return JSON with `{"version": env!("CARGO_PKG_VERSION"), "name": "super-voice", "rust_version": built-in, "features": [...]}`. Use compile-time constants.

3. **GET /api/v1/system/cluster** (`system_cluster`): If runtime_state is available, read the cluster node set from Redis (key pattern `sv:cluster:nodes`). Return JSON array of `{"node_id": str, "address": str, "last_seen": str}`. If Redis not configured, return single-node entry with local info.

4. **POST /api/v1/system/reload** (`system_reload`): Trigger a config reload — if gateway_manager exists, call load_from_config_store(). Return JSON `{"reloaded": true, "timestamp": now}`.

5. **GET /api/v1/system/config** (`system_config`): Return non-sensitive config summary (listening addresses, feature flags, Redis configured yes/no). Do NOT expose Redis URL or API keys.

6. **GET /api/v1/system/stats** (`system_stats`): Return runtime statistics: total_calls, total_failed_calls, active_calls count, uptime, memory usage (if available via std::alloc or jemalloc stats).

Follow existing handler patterns: use `State(state): State<AppState>`, return `impl IntoResponse`, use `Json(serde_json::json!(...))` for responses. Use `require_config_store!` macro where Redis is needed (define locally per module as in other handler files).
