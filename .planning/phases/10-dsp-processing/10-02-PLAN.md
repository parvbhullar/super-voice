<task type="auto">
  <name>Task 1: Wire DSP processors into proxy call dispatch</name>
  <files>src/proxy/dispatch.rs, src/proxy/session.rs, src/proxy/types.rs</files>
  <action>
    Add DSP processor integration to the proxy call path. The approach: when a trunk's media class includes DSP options (echo cancellation, DTMF detection, tone detection, PLC), attach the corresponding SpanDSP processors to the call leg's ProcessorChain.

    1. In src/proxy/types.rs, add a DspConfig struct to ProxyCallContext:
       ```rust
       #[derive(Debug, Clone, Default, Serialize, Deserialize)]
       pub struct DspConfig {
           pub echo_cancellation: bool,
           pub dtmf_detection: bool,
           pub tone_detection: bool,
           pub plc: bool,
           pub fax_mode: Option<String>,  // "terminal" or "gateway" or None
       }
       ```
       Add `pub dsp: DspConfig` field to ProxyCallContext with #[serde(default)].

    2. In src/proxy/dispatch.rs, after loading the trunk config and before creating the ProxyCallSession:
       - Read dsp settings from the trunk's media class config. If the trunk has `media_class` with relevant settings, populate DspConfig. Default: echo_cancellation=true, dtmf_detection=true, plc=true (carrier-grade defaults).
       - Pass the DspConfig through to the session.

    3. In src/proxy/session.rs, when the media bridge is established:
       - If dsp.echo_cancellation is true and feature="carrier", call app_state.stream_engine.create_processor("spandsp_echo") and append to the caller leg's ProcessorChain
       - If dsp.dtmf_detection is true and feature="carrier", create "spandsp_dtmf" processor and append to caller leg
       - If dsp.tone_detection is true and feature="carrier", create "spandsp_tone" processor and append to callee leg (detects far-end tones)
       - If dsp.plc is true and feature="carrier", create "spandsp_plc" processors for both legs
       - If dsp.fax_mode is Some("terminal") and feature="carrier", create "spandsp_fax" processor for the fax leg
       - Gate all SpanDSP integration behind #[cfg(feature = "carrier")] so minimal builds are unaffected

    4. Add a ProxyCallEvent::DtmfDetected { digit: char, timestamp: u64 } variant and ProxyCallEvent::ToneDetected { tone_type: String } variant to types.rs for upstream consumers (CDR, webhooks).

    Ensure all changes compile with both `--features carrier` and without (minimal build).

  <behavior>
    - test_echo_cancellation: Feed a known signal through EchoCanceller with separate tx/rx buffers; verify output has lower energy than input echo (measurable reduction >6dB)
    - test_dtmf_detection_inband: Generate a DTMF tone for digit '5' (770Hz+1336Hz) at 8kHz, feed to DtmfDetector, verify digit '5' is detected and logged
    - test_fax_t38_terminal: Create FaxEngine in terminal mode, process audio frames simulating CNG/CED exchange, verify state machine reaches correct phases
    - test_tone_detection: Generate a busy tone (480Hz+620Hz alternating), feed to ToneDetector, verify ToneType::Busy is detected
    - test_plc_concealment: Feed 10 good frames to PlcProcessor, then simulate loss by calling fill_missing_frame, verify the concealed frame has non-zero energy (not silence) and duration < 60ms gap
    - test_dsp_processor_registration: Verify StreamEngine can create all 5 SpanDSP processors by name
    - test_dsp_config_defaults: Verify DspConfig defaults have echo/dtmf/plc enabled
  </behavior>
  <action>
    Create tests/dsp_integration.rs with #[cfg(feature = "carrier")] gating the entire module:

    1. test_echo_cancellation_measurable_reduction:
       - Create EchoCanceller with tail_len=256 (32ms at 8kHz)
       - Generate a 1kHz sine wave as the "far-end" signal (tx)
       - Create "echo" by attenuating tx by 12dB and delaying 5ms as rx
       - Process 500ms of audio through echo_can_update
       - After convergence (~200ms), measure the echo residual energy in the last 100ms
       - Assert residual is at least 6dB lower than the echo input level
       - This proves SC1 (measurable echo reduction via loopback test signal)

    2. test_dtmf_digit_detected_inband:
       - Create DtmfDetector
       - Generate DTMF digit '5' (770Hz + 1336Hz dual-tone, 40ms duration at 8kHz)
       - Feed the tone + 20ms silence to process_audio()
       - Call get_digits() and assert vec contains '5'
       - Log the digit with chrono::Utc::now() timestamp
       - This proves SC2 (DTMF digit detected inband with digit and timestamp)

    3. test_fax_engine_terminal_mode:
       - Create FaxEngine::new_terminal(true) (calling side)
       - Feed 160-sample (20ms) frames of audio to process_audio()
       - Verify no panic and the engine processes frames
       - If the engine supports state inspection, verify it starts in expected initial state
       - This proves SC3 structure (T.38 terminal mode operates)

    4. test_tone_detection_busy:
       - Create ToneDetector
       - Generate busy tone: 480Hz + 620Hz dual tone, 500ms on / 500ms off pattern at 8kHz
       - Feed 2 seconds of this pattern to process_audio() in 20ms chunks
       - Assert that at least one call returns Some(ToneType::Busy)
       - This proves SC4 concept (tone detection works)

    5. test_plc_concealment_no_silence_gap:
       - Create PlcProcessor
       - Feed 10 good frames of 160 samples (20ms each) of 1kHz sine wave at 8kHz
       - Simulate loss: create a zeroed 160-sample buffer, call fill_missing_frame()
       - Assert the concealed frame has samples where at least some are non-zero
       - Calculate RMS energy; assert it is > 0 (not pure silence)
       - This proves SC4 (packet loss concealed, no >60ms silence)

    6. test_all_processors_registered:
       - Create StreamEngine::default()
       - Assert create_processor("spandsp_dtmf").is_ok()
       - Assert create_processor("spandsp_echo").is_ok()
       - Assert create_processor("spandsp_plc").is_ok()
       - Assert create_processor("spandsp_tone").is_ok()
       - Assert create_processor("spandsp_fax").is_ok()

    - rms_energy(samples: &[i16]) -> f64
