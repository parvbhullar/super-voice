NOTE: The spandsp crate CANNOT depend on active-call (circular dependency). The Processor trait lives in active-call. Instead, spandsp defines its own `SpanDspProcessor` trait that mirrors the interface. The active-call root crate provides adapter wrappers that implement `Processor` for SpanDSP types. Alternatively, extract the Processor trait into a shared crate. The SIMPLEST approach: have the spandsp crate expose a `process_audio(&mut self, samples: &mut [i16]) -> Result<()>` method on each type, and the root active-call crate wraps them in Processor impls. Choose this approach.

2. **src/dtmf.rs** — `DtmfDetector`:
   - Wraps `*mut dtmf_rx_state_t`
   - `DtmfDetector::new() -> Result<Self>`: calls `dtmf_rx_init(ptr::null_mut(), callback, user_data)`
   - C callback trampoline: `extern "C" fn dtmf_callback(user_data: *mut c_void, digits: *const c_char, len: c_int)` — stores detected digits in a `Vec<char>` behind the user_data pointer
   - `pub fn process_audio(&mut self, samples: &[i16]) -> Result<()>`: calls `dtmf_rx(self.state, samples.as_ptr(), samples.len() as c_int)`
   - `pub fn get_digits(&mut self) -> Vec<char>`: drains accumulated detected digits
   - `Drop` calls `dtmf_rx_free(self.state)`
   - `unsafe impl Send for DtmfDetector {}` (per-call, single-threaded access)
   - Static factory function for StreamEngine registration: `pub fn create() -> Result<Box<DtmfDetector>>`

3. **src/echo.rs** — `EchoCanceller`:
   - Wraps `*mut echo_can_state_t`
   - `EchoCanceller::new(tail_len: i32) -> Result<Self>`: calls `echo_can_init(tail_len, 0)`. Typical tail_len = 128 (16ms at 8kHz)
   - `pub fn process_audio(&mut self, tx_samples: &[i16], rx_samples: &mut [i16]) -> Result<()>`: calls `echo_can_update(self.state, tx.as_ptr(), rx.as_ptr())` for each sample pair — note echo_can_update is per-sample, so loop over the frame
   - `Drop` calls `echo_can_free(self.state)`
   - `unsafe impl Send for EchoCanceller {}`
   - Static factory: `pub fn create() -> Result<Box<EchoCanceller>>`

4. **src/tone.rs** — `ToneDetector`:
   - Wraps `*mut super_tone_rx_state_t`
   - `ToneDetector::new() -> Result<Self>`: calls `super_tone_rx_init(...)` with standard tone descriptors
   - `pub fn process_audio(&mut self, samples: &[i16]) -> Result<Option<ToneType>>`
   - `ToneType` enum: `Busy, Ringback, Sit, Unknown`
   - `Drop` calls `super_tone_rx_free(self.state)`
   - `unsafe impl Send for ToneDetector {}`
   - NOTE: ToneDetector initialization is complex (requires tone descriptor setup). If the SpanDSP API for super_tone_rx is too complex for Phase 1, implement as a stub that compiles and returns `Ok(None)` with a TODO comment. The full implementation comes in Phase 10.
   - Static factory: `pub fn create() -> Result<Box<ToneDetector>>`

5. **src/plc.rs** — `PlcProcessor`:
   - Wraps `*mut plc_state_t`
   - `PlcProcessor::new() -> Result<Self>`: calls `plc_init(ptr::null_mut())`
   - `pub fn process_good_frame(&mut self, samples: &mut [i16]) -> Result<()>`: calls `plc_rx(self.state, samples.as_mut_ptr(), samples.len() as c_int)`
   - `pub fn fill_missing_frame(&mut self, samples: &mut [i16]) -> Result<()>`: calls `plc_fillin(self.state, samples.as_mut_ptr(), samples.len() as c_int)`
   - `Drop` calls `plc_free(self.state)`

   - Static factory: `pub fn create() -> Result<Box<PlcProcessor>>`
