       - `ConfigSubscriber` wraps redis PubSub and provides `async fn next_event(&mut self) -> Result<Option<ConfigChangeEvent>>` that deserializes messages

    4. Integration tests with real Redis: one task publishes, another subscribes, assert event received. Use tokio::time::timeout(Duration::from_millis(500)) to enforce latency requirement.
