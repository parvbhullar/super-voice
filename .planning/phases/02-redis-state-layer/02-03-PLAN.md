  <action>
    1. Create src/redis_state/engagement.rs:
       - `pub struct EngagementTracker { pool: RedisPool }`
       - `EngagementTracker::new(pool: RedisPool) -> Self`
       - Redis data model uses two complementary sets per relationship:
         - Forward set "sv:engagement:refs:{source}" — tracks what source references (SADD target)
         - Reverse set "sv:engagement:deps:{target}" — tracks what depends on target (SADD source)
       - `track(source: &str, target: &str) -> Result<()>` — SADD to both forward and reverse sets
       - `untrack(source: &str, target: &str) -> Result<()>` — SREM from both sets
       - `untrack_all(source: &str) -> Result<()>` — Get all members of forward set, SREM source from each reverse set, DEL forward set
       - `check_engaged(target: &str) -> Result<Vec<String>>` — SMEMBERS "sv:engagement:deps:{target}", returns list of sources that reference this target
       - `is_engaged(target: &str) -> Result<bool>` — SCARD > 0 on deps set
       - Entity keys use format "{entity_type}:{name}" e.g., "gateway:gw1", "trunk:us-east"
    2. Re-export from mod.rs
    3. Modify src/redis_state/config_store.rs to integrate EngagementTracker:
       - Add `engagement: Option<EngagementTracker>` field to ConfigStore
       - Add `ConfigStore::with_engagement(self, engagement: EngagementTracker) -> Self` builder method (chainable with with_pubsub)
       - In `set_trunk`: after successful SET, if engagement is Some, call `untrack_all("trunk:{name}")` first (to clear stale refs), then for each gateway in `config.gateways`, call `engagement.track("trunk:{name}", "gateway:{gw_name}")`
       - In `delete_trunk`: if engagement is Some, call `untrack_all("trunk:{name}")` after successful DEL to clean up outgoing references
       - In `delete_gateway`: if engagement is Some, call `engagement.check_engaged("gateway:{name}")` BEFORE the DEL. If non-empty, return `Err` with message like "cannot delete gateway 'gw1': referenced by trunk:us-east, trunk:us-west". Do NOT delete.
       - Same pattern for delete_endpoint: check_engaged before delete
       - In `set_endpoint`, `set_gateway`: no engagement tracking needed (these are leaf resources or targets, not sources of references)
       - For routing_table set: track references to trunks mentioned in routing rules. For routing_table delete: untrack_all. For trunk/gateway delete: check_engaged first.
    4. Integration tests with real Redis: test full lifecycle through ConfigStore — set_trunk with gateway refs, try delete_gateway (should fail), delete_trunk, then delete_gateway (should succeed).
  </action>

       - `pub struct ApiKeyStore { pool: RedisPool }`
       - `ApiKeyStore::new(pool: RedisPool) -> Self`
       - `create_key(name: &str) -> Result<String>` — Generate random 32-byte key, hex-encode to 64-char string, store SHA-256 hash in Redis SET "sv:api_keys" as "{name}:{hash}", return plaintext key prefixed with "sv_" (e.g., "sv_abc123...")
       - `validate_key(key: &str) -> Result<bool>` — Strip "sv_" prefix, SHA-256 hash it, check if any member of "sv:api_keys" ends with that hash
       - `delete_key(name: &str) -> Result<bool>` — Remove matching entry from "sv:api_keys" set
       - `list_keys() -> Result<Vec<String>>` — Return key names (not hashes) from "sv:api_keys"

       - `pub async fn auth_middleware(State(state): State<AppState>, req: Request, next: Next) -> Response`

       - If no header or invalid format: return 401 with JSON `{"error": "unauthorized"}`
       - Validate token against ApiKeyStore
       - If invalid: return 401
       - If valid: call next.run(req).await
       - Skip auth for paths matching configurable skip list (e.g., /health)

    3. Wire middleware into handler/mod.rs: apply auth_middleware as a layer on the carrier API routes (not on existing AI agent routes to avoid breaking them). Create a new router group for carrier admin API that uses the auth layer.
