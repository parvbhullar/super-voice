Create `src/handler/trunks_api.rs` following the exact pattern from `gateways_api.rs`:

**Core trunk CRUD (5 endpoints):**
- `POST /api/v1/trunks` -- create_trunk: validates name non-empty, direction in [inbound, outbound, both], distribution in [weight_based, round_robin, hash_callid, hash_src_ip, hash_destination]. Checks gateways exist via config_store. Returns 201.
- `GET /api/v1/trunks` -- list_trunks: returns all trunks from config_store.list_trunks()
- `GET /api/v1/trunks/{name}` -- get_trunk: returns trunk or 404
- `PUT /api/v1/trunks/{name}` -- update_trunk: full replacement, same validation as create
- `PATCH /api/v1/trunks/{name}` -- patch_trunk: accepts partial JSON (serde_json::Value), merges with existing trunk config. Only updates provided fields. This is key for success criteria #4.
- `DELETE /api/v1/trunks/{name}` -- delete_trunk: returns 204 or 404

**Trunk sub-resource endpoints (13 endpoints):**
Credentials:
- `GET /api/v1/trunks/{name}/credentials` -- list credentials from trunk.credentials
- `POST /api/v1/trunks/{name}/credentials` -- add credential to trunk.credentials vec, save trunk
- `DELETE /api/v1/trunks/{name}/credentials/{realm}` -- remove credential by realm

IP ACL:
- `GET /api/v1/trunks/{name}/acl` -- list ACL entries from trunk.acl
- `POST /api/v1/trunks/{name}/acl` -- add IP/CIDR to trunk.acl vec
- `DELETE /api/v1/trunks/{name}/acl/{entry}` -- remove ACL entry (URL-decode the entry)

Origination URIs:
- `GET /api/v1/trunks/{name}/origination_uris` -- list origination URIs
- `POST /api/v1/trunks/{name}/origination_uris` -- add origination URI
- `DELETE /api/v1/trunks/{name}/origination_uris/{uri}` -- remove origination URI

Media:
- `GET /api/v1/trunks/{name}/media` -- get media config
- `PUT /api/v1/trunks/{name}/media` -- set/replace media config

Capacity:
- `GET /api/v1/trunks/{name}/capacity` -- get capacity config
- `PUT /api/v1/trunks/{name}/capacity` -- set/replace capacity config

All sub-resource handlers: load trunk from config_store, modify the relevant field, save trunk back. Return 404 if trunk not found.

Access config_store from `state.config_store` (it is `Option<Arc<ConfigStore>>`). Return 503 if config_store is None (same pattern as gateways_api).

Create `src/handler/dids_api.rs` with 5 DID CRUD handlers:
- `POST /api/v1/dids` -- create_did: validates number non-empty, trunk non-empty, routing.mode in [ai_agent, sip_proxy]. If mode is ai_agent, playbook should be provided. Returns 201.
- `GET /api/v1/dids` -- list_dids
- `GET /api/v1/dids/{number}` -- get_did or 404
- `PUT /api/v1/dids/{number}` -- update_did (full replacement)
- `DELETE /api/v1/dids/{number}` -- delete_did, returns 204

Use `state.config_store` directly for DID CRUD (ConfigStore already has set_did/get_did/list_dids/delete_did from Plan 01).
