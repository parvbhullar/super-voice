**Success Criterion #4**: "Operator updates trunk max concurrent calls to 5 via PATCH /trunks/{id}; the change is reflected immediately in GET response"
- Build test router using the same pattern as auth.rs tests (AppStateBuilder with Redis)
- Create an API key for auth
- POST /api/v1/trunks to create a trunk with capacity max_calls=100
- PATCH /api/v1/trunks/{name} with body `{"capacity": {"max_calls": 5}}`

**Success Criterion #5**: "Bearer token authentication is enforced on all CRUD endpoints; missing or invalid token returns 401"
- Test each trunk endpoint without auth header -> 401
- Test each trunk endpoint with invalid token -> 401
- Test POST /api/v1/trunks with valid token -> 201 (or appropriate success)

**Trunk sub-resource tests:**
- POST /api/v1/trunks/{name}/acl with `{"entry": "10.0.0.0/8"}` -> 200
- GET /api/v1/trunks/{name}/acl -> returns the ACL list containing "10.0.0.0/8"
- POST /api/v1/trunks/{name}/credentials with credential -> 200
- PUT /api/v1/trunks/{name}/capacity with `{"max_calls": 10, "max_cps": 5.0}` -> 200

Use `tower::ServiceExt::oneshot` for in-process testing (same pattern as existing auth tests).
